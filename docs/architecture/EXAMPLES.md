## Wise Content Factory (WCF)

Version: 1.0Status: Reference Examples & Acceptance TestsOwner: WiseAitechs

## Purpose

This document provides real-world examples that define how Wise Content Factory should behave.
These examples are not demonstrations.
They are acceptance criteria.
If the software cannot complete these examples successfully, the feature should not be considered complete.

## Philosophy

Users should never think about workflows.
They should only think about ideas.
The platform should transform one idea into complete marketing assets.
Every example below should execute with minimal user interaction.

## Acceptance Tests

The platform should pass the following tests.
✓ One prompt generates an entire campaign.
✓ Brand memory loads automatically.
✓ NotebookLM knowledge integrates automatically when available.
✓ Existing assets are reused.
✓ Duplicate generation is avoided.
✓ AI providers are selected automatically.
✓ Costs remain as low as practical.
✓ Multiple brands remain isolated.
✓ Campaigns are optimized for each platform.
✓ Every output is stored and reusable.
✓ Failed tasks recover automatically.
✓ Users never need prompt engineering knowledge.
✓ Users never manually coordinate AI tools.

## Ultimate Example


### User Input

I want to promote PillFill this week.
The platform should automatically:
Understand the objective.
Load the correct brand.
Review previous campaigns.
Retrieve relevant knowledge.
Identify current marketing opportunities.
Develop a campaign strategy.
Generate all required marketing assets.
Package them for publishing.
Present a professional campaign ready for review.
The user should feel like an experienced creative agency completed several days of work after a single conversation.

## Final Statement

The examples in this document define the expected behavior of Wise Content Factory.
Every future feature should be evaluated against these examples.
If a feature makes these workflows simpler, faster, more affordable, or higher quality, it belongs in the platform.
If it increases complexity without providing measurable value, it should not be implemented.
